	# Queue Analytics Service

Веб-приложение для обработки отчётов электронной очереди, объединения входных файлов и построения агрегированной таблицы эффективности сотрудников.

## Что делает приложение

Сервис принимает 4 отчёта в формате CSV/XLSX, нормализует структуру данных, объединяет записи и рассчитывает итоговую сводку по сотрудникам и типам операций.

Результат включает:

- сотрудника
- тип операции
- количество клиентов
- минимальное время
- максимальное время
- среднее время
- категорию эффективности (A / B / C)

Также формируется quality report с информацией о нормализации и качестве входных данных.

## Основные возможности

- загрузка 4 файлов через веб-интерфейс
- поддержка CSV и XLSX
- нормализация названий колонок
- агрегирование по сотруднику и типу операции
- расчёт метрик count / min / max / avg
- классификация по порогам A / B / C
- preview результата в интерфейсе
- скачивание CSV / XLSX / quality report

## Логика расчёта

В рамках решения итоговое время операции рассчитывается как:

`total_time = service_time + processing_time`

Где:

- `service_time` — время непосредственного обслуживания клиента
- `processing_time` — время внутренней обработки операции

Категория определяется по среднему суммарному времени (`avg_time_sec`):

- A — до порога A
- B — до порога B
- C — выше порога B

Пороговые значения задаются пользователем через интерфейс.

## Поддерживаемые входные колонки

Сервис поддерживает нормализацию колонок по словарю синонимов. В базовом варианте ожидаются поля:

- `employee` / `сотрудник`
- `operation_type` / `тип операции`
- `service_time` / `время обслуживания`
- `processing_time` / `время обработки`
- `date` / `дата`
